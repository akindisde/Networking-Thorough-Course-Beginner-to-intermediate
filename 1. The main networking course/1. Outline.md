# 🗺️ Networking Course — Master Outline

## Blueprint: Absolute Beginner → Strong Intermediate

---

## COURSE IDENTITY

Target audience: Complete beginners (zero networking background) Target outcome: Strong intermediate ready for CCNA prep or NSE4 entry Total duration: ~6 months at 4-6 hours/week Format: Video lessons + labs + quizzes + projects Lab tool: Packet Tracer (beginner) → GNS3 (intermediate) Analysis tool: Wireshark (used in every single module)

---

## TEACHING FRAMEWORK (apply to every lesson)

Every lesson follows this exact structure:

1. THE PROBLEM What real-world problem does today's concept solve? Never introduce a concept without this first.
    
2. THE CONCEPT What is it? How does it work? Plain language first. Diagrams drawn live, not shown as pre-made images.
    
3. THE DEMONSTRATION CLI configuration or protocol behavior shown live. Real equipment or GNS3/Packet Tracer.
    
4. WIRESHARK VERIFICATION Show the actual packets. Every protocol, every module. Theory meets reality here — never skip this.
    
5. SECURITY ANGLE How does an attacker see this? What can go wrong? Security is woven in, never left to a separate module.
    
6. HANDS-ON LAB Student builds or troubleshoots something themselves. Watching is not learning. Doing is learning.
    

---

## DESIGN PRINCIPLES

Spiral learning: Introduce a concept simply early, return deeper later. VLANs appear in Module 3, 4, 9, and 12 — getting deeper each time.

Failure-first where possible: Show a broken network before explaining why it is broken. Failure creates the question the lesson then answers.

No magic numbers: 255.255.255.0 is not memorized. It is derived from binary. Every number has a reason the student understands.

Security throughout: ARP → ARP spoofing immediately after. DNS → DNS poisoning immediately after. The student exits thinking like both engineer and attacker.

Gulf-relevant throughout: Fortinet FortiGate configuration (dominant in Gulf enterprise). SD-WAN (Gulf rapidly migrating from MPLS). Design projects set in Saudi/Gulf business context. Arabic documentation mentioned in Module 12.

# MODULE 1 — HOW THE INTERNET ACTUALLY WORKS

Prerequisites: Module 0 Goal: student understands the big picture before any detail

Lesson 1.1 — The Journey of a Web Request - Walk through what happens when you type google.com and press Enter - Cover every step in ONE sentence each (DNS, TCP, TLS, HTTP, render) - Show it live: browser DevTools → Network tab → every request visible - This becomes the outline for the entire rest of the course

Lesson 1.2 — The Protocol Stack and Encapsulation - Why layers exist: separation of concerns, modularity, interoperability - Internet 5-layer model (Application → Transport → Network → Link → Physical) - OSI 7-layer model (why engineers use these numbers in conversation) - Draw encapsulation live: data → segment → datagram → frame → bits - Open Wireshark: show the same structure in a real captured packet - Lab: student identifies all five layers in one Wireshark packet

Lesson 1.3 — Protocols, RFCs, and Standards Bodies - What makes something a protocol (format, order, actions) - IETF and RFCs (TCP, IP, HTTP, DNS, TLS) - IEEE (Ethernet 802.3, WiFi 802.11, 802.1X) - IANA (port numbers, IP allocations) - Key RFC numbers to recognize (791, 793, 768, 826, 1035, 8446)

Lesson 1.4 — Packets, Frames, and Data Flow - Circuit switching vs packet switching (why the Internet uses packets) - Statistical multiplexing: why packet switching serves more users - MTU: what it is, why 1500 bytes, what happens when you exceed it - Packet vs frame: end-to-end vs hop-by-hop, stripped and rebuilt at each router - Bandwidth vs throughput: the pipe diameter vs water flow analogy - Lab: Wireshark — capture HTTP to neverssl.com, identify all layers

Checkpoint quiz: 10 questions on layers, encapsulation, protocols

# MODULE 2 — THE PHYSICAL WORLD

Prerequisites: Module 1 Goal: student understands what carries the bits and the limits of each medium

Lesson 2.1 — Physical Media - Twisted-pair copper: Cat5e / Cat6 / Cat6a / Cat8, 100m limit, why - Fiber optic: single-mode (100+ km) vs multi-mode (550m), WDM - Wireless radio: 2.4 GHz vs 5 GHz, broadcast nature, why encryption is mandatory - Coaxial: cable TV / DOCSIS, shared medium concept - Security property of each: which can be passively tapped, which cannot - Show cable internals: 8 wires, 4 pairs, T-568B wiring order - Lab: students test continuity on a patch cable if hardware available; otherwise trace a cable path in a room

Lesson 2.2 — Signals, Noise, and Attenuation - How bits become signals: voltage (copper), light (fiber), waves (radio) - Attenuation: why signal weakens, why 100m limit exists - Noise and interference: why cables are twisted, what crosstalk is - SNR: signal-to-noise ratio, why higher is better - Latency as a physics problem: speed of light, 45ms Algeria→USA, no solution - Bandwidth (physics) vs latency: two separate things, both matter differently - Lab: ping destinations at different distances, correlate RTT to geography

Lesson 2.3 — Network Devices - Hub: Layer 1, repeats to all ports, collision domain, why obsolete - Switch: Layer 2, reads MAC, forwards to correct port only, CAM table - Router: Layer 3, reads IP, connects different networks, routing table - THE KEY LESSON: same subnet = switch handles it, different subnet = router needed - Access Point: Layer 2 wireless, bridges WiFi and Ethernet - Firewall: inspects and controls traffic between zones - Home router: demystify — it is 5 devices in one box - Lab: Packet Tracer — hub network vs switch network, observe the difference

Checkpoint quiz: 10 questions on media, devices, and their layers

# MODULE 3 — ETHERNET AND SWITCHING (LAYER 2)

Prerequisites: Module 2 Goal: student can design, configure, and secure a Layer 2 LAN

Lesson 3.1 — MAC Addresses - 48-bit hex identifier: structure, OUI (manufacturer), device portion - Special addresses: broadcast (FF:FF:FF:FF:FF:FF), multicast, unicast - Where MAC is used (one hop only) vs where it is not (across routers) - MAC spoofing: what it is, how to do it, why attackers use it - Defense: 802.1X (not MAC filtering — explain why MAC filtering fails) - Lab: find your MAC, look up the OUI, find MACs in Wireshark frames

Lesson 3.2 — How Switches Work — The CAM Table - Learning: switch reads source MAC on arrival, stores port-MAC mapping - Flooding: unknown destination → send to all ports (like a hub temporarily) - Forwarding: known destination → send to correct port only - CAM table aging: why entries expire, what happens when they do - Security: MAC flooding attack (macof), CAM table overflow → switch floods - Defense: port security (maximum MACs per port, violation action) - Config: switchport port-security maximum / violation - Lab: Packet Tracer — show mac address-table before and after pings

Lesson 3.3 — VLANs - The problem: one physical switch, multiple logical networks needed - 802.1Q tag: 4 bytes, VLAN ID field, how it works - Access port: one VLAN, untagged (for end devices) - Trunk port: multiple VLANs, tagged (between switches and routers) - Native VLAN: what it is, why changing it from VLAN 1 matters - Enterprise VLAN template: Management/Servers/Users/Printers/Guest/VoIP - Inter-VLAN routing: Router on a Stick vs Layer 3 switch (SVI) - Security: VLAN hopping attacks (switch spoofing, double tagging), defenses - Config: vlan, switchport mode, switchport access, switchport trunk - Lab: Packet Tracer — two VLANs, test isolation, add router-on-a-stick

Lesson 3.4 — Spanning Tree Protocol - The problem: redundant links → broadcast loops → network meltdown - Demo first: show a loop without STP, watch counters explode - STP solution: elect root bridge, find root ports, block redundant ports - Port states: blocking → listening → learning → forwarding - STP vs RSTP vs PVST+: convergence time comparison - Security: STP manipulation attack (root bridge takeover → MITM) - Defense: BPDU Guard, Root Guard configuration - Config: spanning-tree priority, portfast, bpduguard - Lab: Packet Tracer — three switches in triangle, show STP blocking, simulate link failure

Lesson 3.5 — ARP - The problem: IP layer has destination IP but Ethernet needs destination MAC - ARP process: request (broadcast) → reply (unicast) → cache entry - ARP is local only: for remote destinations, ARP to the default gateway - Gratuitous ARP: what it is, legitimate uses, attack uses - Security: ARP spoofing — no authentication, anyone can claim any IP - The MITM result: attacker intercepts all traffic between victim and gateway - Defense: Dynamic ARP Inspection (DAI), HTTPS everywhere, 802.1X - Config: ip dhcp snooping, ip arp inspection vlan - Lab: Wireshark — filter arp, observe Request and Reply, read cache

Checkpoint: configure a complete small switch with VLANs, trunks, port security, STP hardening. Student submits Packet Tracer file.

# MODULE 4 — IP ADDRESSING AND SUBNETTING

Prerequisites: Module 3 Goal: student subnets any network without a calculator in under 60 seconds

NOTE TO COURSE CREATOR: This module requires MORE practice time than any other. Do not rush through it. Daily subnetting drills must be emphasized in the lesson intro, in the outro, and in the course announcements. Students who skip the practice will fail every practical later.

Lesson 4.1 — Binary: Non-Negotiable Foundation - Why binary: subnet masks ARE binary, understanding them requires binary - The 8 position values: 128, 64, 32, 16, 8, 4, 2, 1 - Binary to decimal: add positions where bit = 1 - The eight magic values: 128, 192, 224, 240, 248, 252, 254, 255 (these are the ONLY values that appear in subnet masks — memorize them) - Decimal to binary: subtract largest fitting power of 2, repeat - Lab: 20 conversion problems, timed. Students keep doing until < 5 sec each.

Lesson 4.2 — IPv4 Addressing - 32-bit number in dotted decimal: what each octet represents - Network portion vs host portion: the fundamental division - Subnet mask: 1s = network, 0s = host, always contiguous - CIDR /prefix notation: count the 1s, write as /N - Private ranges: 10.x.x.x / 172.16–31.x.x / 192.168.x.x - Special: 127.0.0.1 (loopback), 169.254.x.x (APIPA = no DHCP) - Network address, broadcast address, usable host range — derive them - Formula: usable hosts = 2^(host bits) – 2 - Quick reference table: /24 through /30, hosts per subnet - Lab: given 10 random IP/prefix combinations, derive all properties

Lesson 4.3 — Subnetting - Why subnet: performance (reduce broadcast domains), security (isolate traffic) - The 5-step process: subnets needed → bits borrowed → new prefix → block size → list - Block size shortcut: 256 minus the last non-255 octet of the mask - Work through 3 complete examples of increasing complexity live - Is IP X in subnet Y? How to determine quickly - Lab: 20 subnetting problems, timed. Students work down to < 60 seconds each.

Lesson 4.4 — VLSM - The waste problem: equal subnets waste addresses (WAN link needs 2, gets 62) - VLSM rule: always assign LARGEST requirement first - Work through one complete VLSM problem live (5 networks, different sizes) - Verify no overlaps: list ranges, confirm they don't overlap - Lab: design VLSM scheme for a 4-site company (provided brief)

Lesson 4.5 — IPv6 - Why it exists: IPv4 exhausted 2011, NAT breaks end-to-end - 128-bit address, 8 groups of 4 hex digits - Compression rules: remove leading zeros, :: for longest run of zeros - Address types: Global Unicast, Link-Local (auto), Loopback ::1, Unique Local - /64 for LAN subnets, /128 for single hosts - No ARP: replaced by NDP (Neighbor Discovery Protocol) via ICMPv6 - Current status: ~45% of Google traffic (2024), Gulf enterprise dual-stacking - Lab: configure IPv6 on lab VM, ping6, see IPv6 in Wireshark

Lesson 4.6 — DHCP - The problem: manual IP assignment for hundreds of devices - DORA process: Discover → Offer → Request → Acknowledge (explain each) - What a DHCP lease contains: IP, mask, gateway, DNS, lease time - Lease renewal timing: 50%, 87.5%, 100% of lease - DHCP Relay: why needed, how ip helper-address works - Cisco DHCP server config: pool, network, default-router, dns-server, lease - Security: DHCP starvation attack, Rogue DHCP server attack → MITM - Defense: DHCP snooping (ip dhcp snooping + trust uplink only) - Lab: Wireshark — filter dhcp, release and renew, read all four DORA messages

Checkpoint: complete IP addressing plan for a 5-site company using VLSM. Student must work without a calculator. Graded on speed and accuracy.

# MODULE 5 — IP IN DEPTH

Prerequisites: Module 4 Goal: student understands how IP actually works hop-by-hop

Lesson 5.1 — IP Forwarding and TTL - IP is connectionless, best-effort, stateless — what each means - Longest prefix match: when multiple routes match, most specific wins - TTL: what it is, why it exists, what happens when it hits 0 - How traceroute exploits TTL to map the path - Reading traceroute output: each hop, RTT, asterisks (ICMP blocked) - Administrative Distance: trust hierarchy when multiple protocols know same route - Lab: traceroute to 5 different destinations, interpret each hop

Lesson 5.2 — ICMP - ICMP is a helper protocol — not data transport - Key types: Echo Request/Reply (ping), Destination Unreachable, Time Exceeded - Critical teaching point: "ping fails ≠ host is down" (firewalls block ICMP) - Always try connecting to the actual service if ping fails - Security: ping flood, smurf attack (historical), ping sweep for recon - Lab: Wireshark — filter icmp, ping, traceroute, read type and code fields

Lesson 5.3 — NAT - The problem: private IPs not routable, addresses scarce - Three types: Static (one-to-one), Dynamic (pool), PAT/overload (many-to-one) - PAT is what every home router does — explain the port tracking mechanism - How the NAT table maps connections and returns traffic to correct hosts - Port forwarding: DNAT to allow inbound connections to internal servers - Cisco config: ip nat inside/outside, ip nat inside source list … overload - NAT limitations: breaks IPsec, some VoIP protocols, end-to-end connectivity - Lab: Packet Tracer — configure PAT, show ip nat translations during browsing

Lesson 5.4 — MTU and Fragmentation - MTU: maximum packet size per link type (Ethernet = 1500 bytes) - IPv4 fragmentation: router can split, destination reassembles - IPv6: routers CANNOT fragment, source must do it - Path MTU Discovery: DF bit + ICMP Fragmentation Needed response - VPN MTU problem: IPsec adds headers → exceeds 1500 → fragmentation - Symptom: ping works, large downloads fail → MTU issue - Fix: ip mtu on tunnel interface - Lab: ping with -M do -s flags, observe ICMP Fragmentation Needed in Wireshark

Checkpoint quiz: 15 questions on forwarding, ICMP, NAT, MTU

# MODULE 6 — ROUTING

Prerequisites: Module 5 Goal: student configures routing in multi-router environments

Lesson 6.1 — Routing Fundamentals and Static Routing - What a routing table is and how a router uses it - show ip route: reading each line (C, S, O, B — meaning, AD, metric) - Default route: the gateway of last resort, when and why to use it - Static route syntax and configuration - When static is appropriate vs when dynamic routing is needed - Lab: Packet Tracer — 3 routers, configure static routes, test end-to-end

Lesson 6.2 — OSPF - Why OSPF over RIP: bandwidth-based cost, no hop limit, fast convergence - Link-state vs distance-vector: what the difference actually means - OSPF process: neighbor discovery (Hello) → database exchange (LSA) → SPF - OSPF cost: reference bandwidth / interface bandwidth, why to adjust this - Areas: backbone area 0, why areas exist, ABR role - Neighbor states: know what FULL means and what to check if not FULL - OSPF authentication: why to use it, md5 configuration - Config: router ospf, router-id, network, passive-interface - Verify: show ip ospf neighbor, show ip route ospf - Lab: Packet Tracer — 4 routers, OSPF single area, verify, simulate link failure

Lesson 6.3 — BGP Conceptual Understanding - OSPF routes within one AS. BGP routes between ASes. - Autonomous System: what it is, ASN examples (Google = AS15169) - eBGP vs iBGP: between different ASes vs within same AS - BGP path selection attributes at a conceptual level - BGP security: hijacking (no authentication in original design), RPKI as the fix - Students understand WHAT and WHY. Configuration is out of scope here. - Depth: enough to hold a conversation with an ISP engineer

Lesson 6.4 — First Hop Redundancy (HSRP/VRRP) - The problem: single default gateway = single point of failure for all hosts - Virtual IP shared by two routers: hosts never notice the failover - HSRP: active/standby, priority, preempt, virtual IP configuration - VRRP: same concept, open standard, multi-vendor - Config: standby group, ip, priority, preempt - Verify: show standby brief - Lab: Packet Tracer — HSRP, disconnect active, verify failover

Lesson 6.5 — Route Summarization - Why summarize: smaller tables, less protocol traffic, hide topology changes - How to find the summary: find common prefix bits across all routes - Why good VLSM design in Module 4 enables good summarization here - Brief intro to redistribution: when and why (one concept per lesson rule) - Lab: given a set of subnets, find and configure the summary

Checkpoint: multi-router lab from scratch — static + OSPF + HSRP + summary. Student submits working Packet Tracer file.

# MODULE 7 — TCP AND UDP

Prerequisites: Module 6 Goal: student understands transport layer deeply enough to troubleshoot

Lesson 7.1 — Ports and Multiplexing - The problem: multiple applications on one host, one IP address - Port number ranges: well-known, registered, ephemeral - Must-know port numbers: 22, 25, 53, 80, 110, 143, 443, 3306, 3389, 5432 - Socket = IP + Port + Protocol: the complete connection identifier - Lab: ss -tnp to see own open ports, identify what each process is

Lesson 7.2 — TCP Deep Dive - TCP guarantees: reliable, ordered, flow-controlled, congestion-controlled - Three-way handshake: SYN → SYN-ACK → ACK (draw it with sequence numbers) - TCP flags: SYN, ACK, FIN, RST, PSH, URG — when each appears - Sequence and acknowledgment numbers: how TCP tracks every byte - Four-way teardown: FIN → ACK → FIN → ACK - Window size and flow control: receiver tells sender how much it can take - Security: SYN flood attack, SYN cookies defense, TCP RST attack - Lab: Wireshark — follow a TCP stream, find handshake, identify flags, see teardown

Lesson 7.3 — UDP - UDP provides: multiplexing only. Nothing else. - 8-byte header vs TCP's 20+ bytes: what that means for performance - When UDP is the right choice: real-time media, DNS, DHCP, gaming - When TCP is the right choice: everything that must arrive in full - QUIC: HTTP/3 over UDP, why it was created, current adoption - Security: UDP amplification attacks (DNS 40x, Memcached 51,000x) - Lab: Wireshark — filter udp + dns, compare to a TCP stream, count packets

Lesson 7.4 — Congestion Control - Flow control vs congestion control: receiver limit vs network limit - Slow start: why downloads start fast then stabilize - Congestion window: CWND, how it grows and collapses - BDP: Bandwidth × Delay Product — why satellite internet is slow for TCP - Practical implication: why high-latency WAN links need tuning - Depth: conceptual understanding, not algorithm memorization

Lesson 7.5 — Port Scanning with Nmap - What port scanning reveals: open services, versions, OS - Nmap scan types: SYN scan, connect scan, UDP scan, ping sweep - Essential commands: basic, -sV, -O, -A, -p-, -sn - Reading output: open vs closed vs filtered — what each means - ss / netstat: see your own open ports from inside - WARNING: scan only systems you are authorized to scan - Lab: scan virtual lab machines, compare Nmap output to ss output on target

Checkpoint quiz: 15 questions on ports, TCP flags, UDP use cases, congestion

# MODULE 8 — APPLICATION LAYER PROTOCOLS

Prerequisites: Module 7 Goal: student understands the protocols applications use daily

Lesson 8.1 — DNS - Why DNS: humans use names, computers use IPs - DNS hierarchy: root → TLD → second-level → subdomain - Resolution process: recursive resolver → root → TLD → authoritative → answer - Record types: A, AAAA, CNAME, MX, NS, TXT, PTR, SOA — what each does - TTL: caching, why to lower it before DNS migration - Tools: dig (+ trace for full chain), nslookup - Security: cache poisoning, Kaminsky attack, DNSSEC, DoH/DoT, zone transfer - Lab: dig +trace google.com, read every step, find MX, SPF, capture in Wireshark

Lesson 8.2 — HTTP and HTTPS - HTTP request structure: method, path, version, headers, body - HTTP methods: GET, POST, PUT, DELETE, HEAD, OPTIONS - Status code groups + key codes: 200, 201, 301, 302, 400, 401, 403, 404, 500, 503 - Key headers: Host, User-Agent, Cookie, Content-Type, Set-Cookie, Location - HTTP/1.1 vs HTTP/2 vs HTTP/3: what changed and why - HTTPS = HTTP + TLS: confidentiality, integrity, authentication - TLS 1.3 handshake: 1-RTT, certificates, CA role - Security headers: HSTS, CSP, X-Frame-Options - Lab: curl -v http://neverssl.com (see raw HTTP), curl -v https://example.com (see TLS)

Lesson 8.3 — Email Protocols - Architecture: MUA → MTA → MTA → MDA - SMTP (25/587), IMAP (143/993), POP3 (110/995) — role of each - IMAP vs POP3: server-side vs client-side, when to use each - Manual SMTP: telnet to port 25, EHLO, MAIL FROM, RCPT TO, DATA - Security: SMTP has no authentication → email spoofing is trivial - SPF, DKIM, DMARC: the three defenses, how they work together - Lab: read full email headers, find SPF/DKIM results, dig TXT for SPF record

Lesson 8.4 — SSH - Why SSH replaced Telnet: plaintext vs encrypted - SSH connection: TCP 22 → key exchange → host key verification → auth - known_hosts: why it matters, what the REMOTE HOST CHANGED warning means - Key-based authentication: keygen, copy to server, auth without password - Hardening: PermitRootLogin no, PasswordAuthentication no, fail2ban - Lab: set up SSH, connect with password, generate keys, connect with key, disable password auth

Lesson 8.5 — FTP, TFTP, and Secure File Transfer - FTP: active vs passive mode (why passive is always preferred with NAT) - FTP is plaintext: demo credential capture in Wireshark - Alternatives: SFTP (SSH-based), FTPS (TLS-based), SCP - TFTP: port 69 UDP, no auth, used for PXE boot and device config transfer - Lab: FTP session captured in Wireshark (see credentials), then SFTP (encrypted)

Checkpoint: configure DNS, test HTTP with curl, set up SSH with key auth, demonstrate email header reading

# MODULE 9 — NETWORK SECURITY FUNDAMENTALS

Prerequisites: Module 8 Goal: student applies security thinking to every network element

NOTE: Security has appeared in every module so far. This module goes deeper and builds a unified security framework.

Lesson 9.1 — Firewalls - What a firewall does: permit or deny based on rules, default deny is correct - Stateless vs stateful: why stateless requires explicit return rules - NGFW: application awareness, user identity, SSL inspection, IPS - Security zones: Outside, DMZ, Inside, Management — traffic rules between each - DMZ design: what goes there, why it cannot initiate to internal - Cisco ACL: extended ACL syntax, rule order (first match wins), deny-all at end - FortiGate policy: srcintf/dstintf/srcaddr/dstaddr/service/action/utm-status - Lab: configure ACL — allow HTTPS and DNS, block everything else, verify

Lesson 9.2 — VPN - Site-to-site vs remote access: use cases for each - Full tunnel vs split tunnel: tradeoffs - IPsec: transport vs tunnel mode, IKE Phase 1 and Phase 2 - AH vs ESP: always use ESP (AH provides no encryption) - Strong algorithms: AES-256-GCM, SHA-256+, DH Group 14 minimum - Weak algorithms to avoid: DES, 3DES, MD5, DH Groups 1/2/5 - FortiGate IPsec config: phase1-interface and phase2-interface - Lab: Packet Tracer — site-to-site IPsec, Wireshark before/inside VPN

Lesson 9.3 — IDS and IPS - IDS detects and alerts (passive). IPS detects and blocks (inline). - Signature-based: known attacks, low false positive, blind to zero-days - Anomaly-based: baseline deviation, can catch zero-days, high false positive - Policy-based: explicit policy violation alerting - NIPS vs NIDS vs HIDS: placement and what each sees - SSL inspection problem: 85% of traffic is HTTPS, IPS cannot see inside - SSL inspection solution: FW decrypts, inspects, re-encrypts

Lesson 9.4 — Network Hardening - Principle: least privilege applied to every device and service - Router checklist: SSH only, strong passwords, disable unused services, no CDP, OSPF authentication, centralized logging - Switch checklist: explicit access ports, port security, BPDU Guard, DHCP snooping, Dynamic ARP Inspection, shut unused ports - Password policy: 16+ chars, unique per device, changed on staff departure - AAA: RADIUS or TACACS+ for centralized network device authentication - Lab: apply hardening checklist to a router and switch, verify each change

Lesson 9.5 — Monitoring and Logging - Cannot secure what you cannot see: logging is not optional - Syslog: severity levels 0-7, what to log at each, centralize everything - SNMP v3: why v1/v2c are insecure, v3 auth + encryption - NetFlow: traffic metadata (not content), use for anomaly detection - What to monitor: auth failures, config changes, interface events, new devices - SIEM: correlate events from all sources, alert on patterns - Lab: configure syslog on router, trigger events, see them arrive at Kali

Checkpoint: security audit of a provided configuration. Find and fix 5 security weaknesses. Justify each fix.

# MODULE 10 — WIRELESS NETWORKING

Prerequisites: Module 9 Goal: student designs, secures, and troubleshoots WiFi

Lesson 10.1 — WiFi Standards and Radio Concepts - 802.11 evolution: b, g, n, ac (WiFi 5), ax (WiFi 6), be (WiFi 7) - 2.4 GHz: range, interference, only 3 non-overlapping channels (1, 6, 11) - 5 GHz: speed, less interference, many non-overlapping channels - 6 GHz: WiFi 6E and 7, no legacy interference, indoor - Channel interference: why adjacent channels overlap, how to avoid it - Signal strength: dBm scale, -30 to -90 range and what each means

Lesson 10.2 — WiFi Security - WEP: broken completely, never use (show why in 30 seconds) - WPA: TKIP over RC4, phase out all WPA devices - WPA2: AES-CCMP, Personal (PSK) vs Enterprise (802.1X + RADIUS) - WPA2 vulnerability: offline dictionary attack on captured handshake - WPA3: SAE (resists offline attack), PMF (mandatory, blocks deauth), OWE - Enterprise WiFi: Thin APs + WLC, CAPWAP, central policy, roaming - WiFi attacks: Evil Twin, Deauthentication (WPA2 unprotected management frames) - Lab: WiFi analyzer app on phone — scan area, identify channels, signal strength

Lesson 10.3 — Wireless Network Design - Site survey: what to look for, tools (Ekahau, NetSpot), documentation - AP placement: 15-20% overlap for roaming, ceiling mount preferred - Channel plan: 2.4 GHz (1/6/11 rotation), 5 GHz (more flexibility) - Capacity: 25-30 active clients per AP max - Guest WiFi: isolated VLAN, internet only, captive portal, bandwidth limit - Band steering: push dual-band clients to 5 GHz - Lab: design WiFi for a provided floor plan (draw.io)

Lesson 10.4 — WiFi Troubleshooting - Problem categories: cannot connect, connected but slow, intermittent drops - Decision tree for each category - Signal strength thresholds: -70 dBm minimum for reliable operation - Channel congestion: how to identify and fix - Diagnostic commands: iwconfig, iwlist scan, iw dev link - Lab: given a described WiFi problem, diagnose using the decision tree

Checkpoint quiz: 12 questions on standards, security, design, troubleshooting

# MODULE 11 — WIDE AREA NETWORKS

Prerequisites: Module 10 Goal: student understands how organizations connect sites and the internet

Lesson 11.1 — WAN Technologies - Leased lines: dedicated, guaranteed, expensive (T1/E1) - MPLS: ISP-managed private WAN, label switching, QoS guarantees - Internet VPN: cheap, less predictable, good enough for most - SD-WAN: multiple links + intelligent steering + automatic failover Show cost comparison: MPLS SAR 15,000/month vs SD-WAN SAR 3,000/month - Cellular WAN: 4G/5G as primary or backup - Why the Gulf is migrating from MPLS to SD-WAN rapidly

Lesson 11.2 — ISP Connectivity - Single ISP: simple, single point of failure - Dual ISP: redundancy and load balancing - BGP multi-homing: full routing control for large organizations - CDN: how content delivery networks reduce latency - IXPs: what they are, major ones, UAE-IX/Gulf context - Lab: traceroute to various destinations, identify ISP hops and peering points

Lesson 11.3 — SD-WAN Deep Dive - Architecture: edge device, controller, underlay, overlay - Link quality monitoring: latency, jitter, loss measured continuously - Traffic steering policies: match app type to link quality requirement - FortiGate SD-WAN config: members, health-check, performance SLA, service rules - Show a steering decision: voice goes to low-latency link, backup to cheap link - Lab: FortiGate VM — configure two WAN members, health check, one steering rule

# MODULE 13 — TROUBLESHOOTING METHODOLOGY

Prerequisites: Module 12 Goal: student diagnoses network problems systematically

Lesson 13.1 — The Troubleshooting Framework - First question: "What changed recently?" (80% of problems follow a change) - Scope the problem: one user / one VLAN / one site / everyone - Three approaches: bottom-up, top-down, divide and conquer — when to use each - The 8-step process: define → gather → analyze → eliminate → theorize → test → implement → verify and document - Essential command toolkit: ip addr, ip route, ping, traceroute, dig, arp -n, ss -tnp, curl -v, tcpdump/Wireshark

Lesson 13.2 — Common Scenario Walkthroughs - "Cannot access the internet": step-by-step decision tree - "VPN connects but cannot reach internal resources": step-by-step - "Users in VLAN 10 cannot reach VLAN 20": step-by-step - "Switch port will not come up": step-by-step - "Website loads slowly": step-by-step - For each: exact commands at each step, what to look for, what it means

Lesson 13.3 — Wireshark as a Troubleshooting Tool - Capture filters: reduce what is recorded (host, port, net, not port) - Display filters: the essential toolkit (ip.addr, tcp.port, tcp.flags.syn, tcp.analysis.retransmission, dhcp, dns, arp) - Diagnosing: cannot connect (SYN with no SYN-ACK), slow page (retransmissions), DHCP failure (Discover with no Offer) - Follow TCP Stream: see full conversation, essential for HTTP diagnosis - Statistics: IO Graph for volume, Conversations for busiest pairs - Lab: deliberate misconfiguration lab — student diagnoses using only Wireshark + ping/traceroute/dig (no show commands)

Checkpoint: given 5 broken configurations, find and fix all 5. Document the diagnosis process for each.
